Cybersecurity Incident Report
Attack Identification:
The evidence from the packet capture logs indicates multiple IP addresses sending a large
number of SYN packets. This suggests a SYN Flood attack, which is a form of Distributed
Denial of Service (DDoS) attack.
Attack Analysis:
The attacker is performing a SYN ﬂood attack, which exploits the TCP handshake process.
Normally, a client sends a SYN packet to initiate a connection, the server responds with a
SYN-ACK, and then the client completes the handshake with an ACK. In this attack, the
malicious devices ﬂood the server with SYN packets but do not respond to the SYN-ACK
replies, leaving half-open connections. This overloads the server's resources, causing it to
become unresponsive or crash due to the incomplete connections.
Mitigation:
To mitigate this, I temporarily blocked the offending IP addresses.
This is not a long-term solution, as the attacker can spoof new IP addresses. A more
effective measure would involve conﬁguring the ﬁrewall to limit the number of SYN-ACK
packets allowed before considering the connection attempt invalid.
Additionally, analysis of the packet logs shows repeated attempts to access /sales.html
ﬁle. This shows the attacker was interested in data reconnaissance of the sales
infrastructure.
