Sneaker company  
The application has several critical business objectives: ensuring  
user data privacy to build customer trust, providing a seamless  
user experience for account management and communication  
between buyers and sellers, and implementing secure, PCI  
DSS-compliant payment handling to avoid legal and financial  
issues. These objectives are essential to achieving the company's  
goal of creating a reliable and efficient marketplace for sneakers.  
I. Deﬁne business  
and security  
objectives  
Securing the SQL database is a top priority because it stores  
sensitive user data, including passwords and payment details,  
making it a prime target for SQL injection attacks if not properly  
secured. Additionally, protecting the Public Key Infrastructure (PKI)  
is critical to ensure the safe exchange of encryption keys,  
preventing potential data breaches during transmission.  
II. Deﬁne the  
technical scope  
When a user searches for a product, the application processes the  
input and sends a query to the database to retrieve relevant  
inventory. To ensure security during this process, user input must  
be validated and sanitized to prevent SQL injection attacks. The  
application uses PKI encryption to secure data in transit,  
protecting against man-in-the-middle attacks. Results returned  
from the database are displayed securely to the user, ensuring  
sensitive data (e.g., personal identifiers) is excluded unless  
necessary. The process is logged for monitoring, enabling the  
detection of unusual or malicious queries."
