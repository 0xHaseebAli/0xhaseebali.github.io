Security incident report  
Section 1: Identify the network protocol involved in the incident  
DNS (Domain Name System): The protocol used to resolve domain names  
(`yummyrecipesforme.com` and `greatrecipesforme.com`) into IP addresses.  
HTTP (Hypertext Transfer Protocol): The protocol used to send and receive  
webpage data between the client (browser) and the web server.  
TCP/IP (Transmission Control Protocol/Internet Protocol): The underlying  
protocol suite that manages the data transmission for DNS and HTTP traffic.  
Section 2: Document the incident  
Incident Summary:  
A former employee, who had access to the company, executed a brute force  
attack to gain access to the admin panel of the website  
`yummyrecipesforme.com`. The attacker guessed the password due to weak  
security settings, specifically the use of a default password, and logged into the  
admin panel.  
Once logged in, the attacker embedded malicious JavaScript code into the  
website’s source code. This code prompted visitors to download a malicious  
file as a browser update. Once visitors downloaded and executed the file, they  
were redirected to a malicious website (`greatrecipesforme.com`).  
Timeline of Events:  
1. DNS Request for `yummyrecipesforme.com`:  
The browser initiated a DNS request to resolve the domain name  
`yummyrecipesforme.com` into an IP address.  
2. DNS Reply:  
The DNS server responded with the correct IP address for  
`yummyrecipesforme.com`.  
3. HTTP Request to `yummyrecipesforme.com`:  
The browser initiated an HTTP request to load the webpage of  
`yummyrecipesforme.com`.  
4. Malware Download:  
The website prompted the user to download a malicious executable file,  
faking it as a browser update.  
5. DNS Request for `greatrecipesforme.com`:  
After running the downloaded file, the browser initiated another DNS request  
to resolve the domain `greatrecipesforme.com`.  
6. DNS Reply:  
The DNS server responded with the correct IP address for  
`greatrecipesforme.com`.  
7. HTTP Request to `greatrecipesforme.com`:  
The browser was redirected to the malicious website containing malware.  
Section 3: Recommend one remediation for brute force attacks  
Recommendation: Enforce Strong Passwords and Two-Factor Authentication  
(2FA)  
Why? A strong password policy requires users to create complex passwords,  
making it more difficult for attackers to guess or brute force them. Enforcing  
2FA adds an extra layer of security by requiring a second form of  
authentication, such as a one-time code sent to a mobile device. Even if an  
attacker guesses the password, they would still need access to the second  
factor to log in.  
Additional Measures:  
Implement rate limiting to restrict the number of login attempts.  
Regularly change passwords and disallow previously used passwords.  
Monitor login activity and flag suspicious attempts for further investigation.
