Security Hardening Recommendations Report  
Tools and Methods:  
Password Policy: Enforce complexity rules and 90-day rotation cycles.  
Firewall Configuration: Filter malicious traffic, log network activities, and  
block unused ports.  
Multi-Factor Authentication (MFA): Restrict sensitive data access to verified  
authorized users only.  
Recommendations & Justification:  
Password Policy:  
Changing passwords regularly and enforcing complexity makes it harder for  
threat actors to carry out brute force attacks. Simple passwords are easy to  
crack, but enforcing complex password rules (such as a mix of characters,  
numbers, and symbols) increases security.  
Firewall Configuration:  
Configuring firewalls to filter traffic strengthens network segmentation and  
protects against attacks like DoS and DDoS. Firewalls can log all incoming and  
outgoing traffic, making it easier to investigate incidents post-attack. Disabling  
unused ports through firewalls also reduces the network's attack surface,  
preventing unauthorized access.  
Multifactor Authentication (MFA):  
MFA improves security by requiring multiple forms of verification, such as a  
one-time code or biometric confirmation, in addition to a password. This  
ensures that even if credentials are compromised through phishing or brute  
force, unauthorized access remains blocked without the second factor.  
Additional Measures:  
To further enhance security, disabling unused applications and services helps  
reduce vulnerabilities. Regularly applying security patches ensures the system  
is protected against known vulnerabilities. Setting clear network access  
privileges limits the exposure of sensitive data. Daily firewall maintenance is  
essential to ensure rules are up-to-date. Finally, proper disposal of hardware is  
critical, as attackers could recover data from improperly discarded devices.
